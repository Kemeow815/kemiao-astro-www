## 人形机器人

从去年开始，A 股市场上人形机器人概念不时地火起来，炒作一波接着一波，靠的是什么呢？靠的是不知所以的各种概念、工作室精心调教过的视频。

这周有个新闻闯入了我的视野，“人机共跑半程马拉松”在北京亦庄开跑，20 支人形机器人参加了比赛。我觉得很抽象，甚至有点搞笑。人形机器人跑步的样子就像是一个人穿着一身黑色的衣服，里面装着一堆电线和电池，走路的时候还要小心翼翼地保持平衡。它们的身后还配置了 2-3 个工作人员，我不知道人类工作人员的职责是干什么的，是在旁边给人形机器人加油，还是在旁边看着人形机器人摔倒了赶紧扶起来？和我之前看的工作室放出的视频不一样啊，既然作为人工智能人形机器人，如果连跑步都无法独立完成，那么它们存在的意义是什么呢？

也许我是错的，就像前几年看人工智能写文章、画画一样。短短 3 年时间，写文章、写代码、画画的人工智能都已经发展得越来越成熟。未来人形机器人也许也会突飞猛进，成为我们生活中不可或缺的一部分。但不是今年。

## 技术速递

- [Introducing GPT-4.1 in the API](https://openai.com/index/gpt-4-1/)——OpenAI 发布了 GPT4.1 系列模型，包括：GPT-4.1、GPT-4.1 mini 以及 GPT-4.1 nano。仅供通过 API 调用。
- [Gemini Advanced is free for college students through finals 2026](https://gemini.google/students/)——谷歌的一项推广活动，Gemini Advanced 模型免费提供给大学生使用，截止到 2026 年的期末考试。（仅限美国大学）
- [MarkItDown-MCP](https://github.com/microsoft/markitdown/tree/main/packages/markitdown-mcp)——MarkItDown 是微软推出的使用 Python 将 Office 文档、URL、音频等转换为 Markdown 的工具。本周他们发布了 MCP 版本。

---

GPT-4.1 详情：
- GPT‑4.1：整体性能大幅提升，尤其在编码任务和复杂指令遵循上表现优异。支持百万级上下文窗口。
- GPT‑4.1 mini：智能体量虽小，但性能在很多标准测试中甚至超过了 GPT‑4o，同时具有更低的延迟和成本。
- GPT‑4.1 nano：主打低延迟与低成本，适合需要快速响应的任务，如自动补全或分类任务，且同样支持长达 1 百万令牌的上下文窗口。

---

MarkItDown-MCP 支持 STDIO 和 SSE 两种方式：

```shell
pip install markitdown-mcp

# STDIO
markitdown-mcp

# SSE
markitdown-mcp --sse --host 127.0.0.1 --port 3001
```
